# pickup

이전 handoff 폴더를 골라 `INDEX.md` 부터 읽고, 가져온 내용 요약만 사용자에게 보여준다. 사용자가 명시적으로 지시할 때까지 작업은 진행하지 않는다.

---

## 동작 순서

### 1. 후보 스캔
`.agent-memory/handoff/` 직속 폴더 목록을 조회. `_archive/` 는 제외.

- 폴더명 사전식 정렬만으로 최신 판별 (타임스탬프가 접두어이므로 시간순과 일치)
- 심볼릭 링크·포인터 파일은 사용하지 않음 (크로스 플랫폼 호환)

### 2. 선택
- 0 개 → 이어받을 handoff 없음을 사용자에게 알리고 종료
- 1 개 → 즉시 로드 (사용자 확인 생략)
- 2 개 이상 → 후보 목록을 최신순으로 제시하고 사용자에게 선택 요청

#### 후보 표시 형식
```
1. 2026-04-25T1610_new-work
2. 2026-04-25T1430_handoff-plugin-design
```

번호로 선택받거나, 사용자가 이름의 일부를 지정하면 부분 일치로 매칭.

### 3. 로드
- 선택된 폴더의 **`INDEX.md`를 먼저** 읽는다
- INDEX의 진입점 안내와 의존관계에 따라 작업 문서(`NN-*.md`)를 순회
- 읽은 내용을 현재 세션의 작업 기반으로 삼는다

### 4. 아카이브
로드 성공 확인 후 해당 폴더를 `.agent-memory/handoff/_archive/`로 이동.

- 목적: 재호출 시 같은 handoff가 후보에 반복 등장하지 않도록
- 실패·중단 시 이동하지 않음 (원자성)
- `_archive/` 디렉토리가 없으면 생성
- 명령 예: `mv .agent-memory/handoff/<folder> .agent-memory/handoff/_archive/`

---

## 다음 할 일 응답 규칙

사용자가 별도 지정 없이 "다음 할 일", "이어서 뭐 해", "어디서부터" 같은 식으로 물으면, **INDEX.md 의 "현재 컨텍스트" 로 지정된 파일의 "다음 수" 만 출처로 삼는다**.

- `[done]` 으로 표시된 다른 작업 파일에 "다음 수" 가 보여도 이미 종결된 일이므로 제안하지 않는다.
- 현재 컨텍스트가 `없음 (세션 종결)` 이면 "이어받을 활성 작업 없음" 으로 답하고, 새 지시를 사용자에게 요청한다.
- 현재 컨텍스트가 누락된 (구버전) handoff 라면 사용자에게 어느 파일을 이어갈지 묻는다 — 추측하지 않는다.

pickup 직후 자동 실행 금지 (서두 원칙) 와 함께 작동: 요약만 보여주고, 사용자가 명시적으로 "다음 할 일" 을 물어야 비로소 위 규칙으로 답한다.

## 재로드 정책

아카이브된 handoff는 원칙적으로 후보에서 제외한다. (사용자가 직접 언급하는 경우 제외)

- 두 스킬 외의 세션 연속성 메커니즘과는 간섭하지 않는다
