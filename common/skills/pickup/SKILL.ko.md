# pickup

이전 handoff 세션의 맥락을 읽어 현재 세션에서 이어받는다. `handoff` 스킬의 복원 짝.

---

## 동작 순서

### 1. 후보 스캔
`.agent-memory/handoff/` 직속 폴더 목록을 조회. `_archive/`는 **제외**.

- 폴더명 사전식 정렬만으로 최신 판별 (타임스탬프가 접두어이므로 시간순과 일치)
- 심볼릭 링크·포인터 파일은 사용하지 않음 (크로스 플랫폼 호환)

### 2. 선택
- **0개** → 이어받을 handoff 없음을 사용자에게 알리고 종료
- **1개** → 즉시 로드 (사용자 확인 생략)
- **2개 이상** → 후보 목록을 **최신순**으로 제시하고 사용자에게 선택 요청

#### 후보 표시 형식
```
1. 2026-04-25T1610_new-work
2. 2026-04-25T1430_handoff-plugin-design
```

번호로 선택받거나, 사용자가 이름의 일부를 지정하면 부분 일치로 매칭.

### 3. 로드
- 선택된 폴더의 **`INDEX.md`를 먼저** 읽는다
- INDEX의 진입점 안내와 의존관계에 따라 작업 문서(`NN-*.md`)를 순회
- 읽은 내용을 현재 세션의 작업 기반으로 삼는다

### 4. 아카이브
로드 성공 확인 후 해당 폴더를 `.agent-memory/handoff/_archive/`로 이동.

- 목적: 재호출 시 같은 handoff가 후보에 반복 등장하지 않도록
- 실패·중단 시 이동하지 않음 (원자성)
- `_archive/` 디렉토리가 없으면 생성
- 명령 예: `mv .agent-memory/handoff/<folder> .agent-memory/handoff/_archive/`

---

## 재로드 정책

아카이브된 handoff는 원칙적으로 후보에서 제외한다. 다시 꺼내볼 필요가 있을 땐:
- 사용자가 경로를 직접 지정하거나
- `_archive/`에서 원 위치로 수동 이동 후 `/pickup` 재호출

pickup 자체는 아카이브를 자동 탐색하지 않는다.

---

## 경계

- **handoff** — 세션 맥락을 저장하는 짝
- **pickup** — 저장된 맥락을 불러오는 짝
- 두 스킬 외의 세션 연속성 메커니즘(예: MEMORY.md, notepad)과는 간섭하지 않는다
