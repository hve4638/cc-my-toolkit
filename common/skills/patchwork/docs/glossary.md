# 용어 사전

patchwork 스킬 문서들이 공유하는 어휘.

## 스킬의 성격

- **patchwork / 누더기 / 조각보** — 이 스킬의 이름이자 성격. 완결된 체계가 아니라 필요할 때마다 조각을 덧대며 자라는 구조.
- **가드레일** — 강제 없이 경계만 표시하는 장치. 프레임워크의 반대 지향.
- **두 축** — patchwork 원칙의 두 계열. 맥락 보존과 독립 검증.
- **맥락 보존** — "맥락은 새어나간다" 라는 관찰에 대응하는 원칙 계열. `docs/10-context/`.
- **독립 검증** — "자기 자신은 못 믿는다" 라는 관찰에 대응하는 원칙 계열. `docs/20-verification/`.
- **구체화 예정** — 각 문서 하단의 열린 질문 섹션. 사용자 결정 전까지 선제 구체화 금지.

## 모듈

- **모듈** — 블랙박스 취급의 단위. 라이브러리 자체일 수도, 소스코드 몇 개가 담긴 디렉토리일 수도 있다.
- **블랙박스** — 특정 단위부터 그 내부를 열지 않고 외부 계약으로만 다루는 방식.
- **진입점** — 모듈의 외부 경계가 되는 파일. 상세한 주석과 파일 링크를 담는다.

## 용어와 주석

- **바운디드 컨텍스트** — 모듈마다 고유의 용어 체계를 갖는다는 규칙. DDD에서 차용.
- **Ubiquitous Language** — 모듈 내에서 하나의 개념에 하나의 이름만 사용한다는 규칙. DDD에서 차용.
- **의도 주석** — "왜 이렇게 하였는가"를 담은 주석.
- **롤백** — 이전에 변경한 코드를 과거의 문제 있던 상태로 되돌리는 행위.
- **롤백 감시** — 의도 주석과 평가자를 연결하는 예외 처리 구조 (구체화 예정).

## 검증

- **작성자** — 코드 작업을 수행하는 주체.
- **평가자** — 작성자와 별도로 결과물을 평가하는 주체. subagent 또는 codex.
- **claude-codex** — Claude와 Codex의 쌍 구조. 이 맥락에서는 평가 및 검증의 지표로만 사용.
- **치팅** — 에이전트가 평가를 우회하는 행위.

## 병렬

- **작업 병렬** — 서로 다른 작업을 여러 서브에이전트가 나누어 수행. 금지.
- **관점 병렬** — 같은 산출물을 여러 모델이 교차검증하는 구조. 허용.
