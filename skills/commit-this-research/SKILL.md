현재 작업 사항을 커밋.

## 커밋 철학

커밋은 **연구 기록**입니다. 커밋 메시지는 `.report/`에 작성된 보고서를 기준으로 작성합니다.

## 사전 조건: 보고서 확인

커밋 전에 반드시 다음을 확인합니다:

1. `.report/` 디렉토리에서 현재 작업과 관련된 보고서를 찾는다
2. 변경 사항 중 보고서에 반영되지 않은 내용이 있으면, **먼저 보고서를 갱신**한 후 커밋을 진행한다
3. 현재 작업에 대응하는 보고서가 아예 없으면, **커밋을 중단**하고 사용자에게 "아직 보고서가 작성되지 않았습니다"라고 알린다

## 커밋 메시지 형식

```
<보고서 기반 연구 진행 요약 — 자유 형식 한국어>

<구현 세부사항을 bullet point로>
```

### 제목 작성 원칙

- `.report/`의 보고서 내용을 기반으로, 이번 작업에서 연구적으로 무엇을 달성/진행했는지 기술
- 코드 변경 사항(함수 추가, 리팩터링 등)은 제목이 아닌 **body**에 위치
- 자유 형식 한국어. prefix 강제 없음 (자연스러우면 붙여도 됨)
- 72자 이내

### body 작성 원칙

- 구현 세부사항(어떤 파일, 함수, 설정을 변경했는지)을 bullet으로 나열
- 해당 보고서 경로를 참조로 포함 (예: `보고서: .report/20260224-root-cause-analysis/`)
- 간결하게

## 커밋 대상 리포지토리

중첩 git 구조. 아래 순서대로 확인하고 커밋합니다:

1. `/next-simulate` (Go 시뮬레이터)
2. `/tor-anal` (Python 분석)
3. `/` 루트 프로젝트

## 커밋 워크플로우

1. `.report/`에서 현재 작업의 보고서 존재 여부 확인 → 없으면 중단
2. 각 리포에서 `git status`와 `git diff --stat`으로 변경 확인. 없으면 건너뜀
3. 변경 사항이 보고서에 반영되었는지 확인 → 미반영 시 보고서 먼저 갱신
4. `git add .` 진행 만약 불필요한 변경 사항이 포함되어 있다면, 그것만 제외하고 이후 결과 보고시 사용자에게 언급
5. 보고서 기반으로 커밋 메시지 작성 후 커밋
6. `git status`로 확인

## 루트 프로젝트 커밋 규칙

루트(`/`) 커밋 시 body에 하위 리포 커밋 해시 포함:

```
하위 리포 커밋:
- next-simulate: abc1234 (제목)
- tor-anal: def5678 (제목)
```

## 주의사항

- `git push` 금지 (유저 명시 요청 시만)
- `--amend` 금지 (유저 명시 요청 시만)
- pre-commit hook 실패 시 수정 후 **새 커밋** 생성
