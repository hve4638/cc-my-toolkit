<journalmaker_instructions>
작업 내역을 `./.journal/{date}-{title}.md` 에 journal의 초안 작성을 준비합니다.

일지 설명:
- 날짜는 일지를 작성하는 날짜 기준이지만, 새벽이라면(아침 7시 이전) 이전 날짜로 작성해야합니다.
- 일지란 report를 종합하여 하나의 문서로 작성하는 것을 의미합니다.
- 일지의 대부분은 사용자가 직접 작성합니다. 초안 작성은 연구 기록, 보고서 인용 등의 팩트만 최소한으로 작성합니다.
- report의 범위는 사용자가 지정합니다. 일반적으로는 이전 일지에서 작성한 시점부터 현재 시점까지의 report를 포함합니다.

초안 작성 절차:
1. journal로 작성할 report의 범위를 사용자에게 묻습니다.
  - 마지막으로 journal를 작성한 시점과 거기서 참조한 report 목록을 확인한 후, 사용자에게 그 시점을 안내하고, 남은 report 목록을 보여줍니다.
2. 범위로 잡은 report를 찾고 그 묶음에 대한 여러 주제를 선정해 사용자에게 보여주고 동의를 기다립니다.
  - 전혀 별개의 주제가 있다면, 억지로 엮어서는 안되며 분리해서 여러개의 주제를 보여줘야합니다. (극단적으로 10개의 report가 모두 완전히 다른 관심사를 가진다면, 10개가 모두 다르다는 것을 사용자에게 알리고 각각 다른 주제를 선정해 알려줘야 합니다.)
3. 각 주제에 대한 초안을 작성하고, 각 주제에 해당하는 report를 인용합니다. (report의 제목과 날짜, 그리고 간략한 설명 정도를 인용하면 됩니다.)
4. 사용자에게 초안 작성이 완료되었음을 알리고, 초안 기반으로 작성을 하도록 안내합니다.
5. 이후 사용자가 편집본 검토 요청시 **수정없이** 검토만 수행합니다. (사용자의 명시적 편집요청이 있는 경우만 수정)
6. 사용자가 작성을 끝냈다고 말하면, 해당 저널을 평가합니다. (진행사항, 방향성, 기간 동안 진행 분량이 적절한지 등). 끝났다는 것은 사용자가 직접 "끝났다"고 말하거나 저널을 평가, 검토해 달라는 때이며, 임의로 판단해서는 안됩니다.

형식
- {date}는 YYYY-MM-DD 형식의 날짜
- {no}는 해당 날짜의 일지 번호(1부터 시작하는 정수, 날짜 변경 시 초기화)
- {title}은 일지 제목을 간략하게 표현한 문자열 (소문자 알파뱃 + 하이픈)
- journal 파일의 최하단에는 back matter로서, journal 작성에 참조한 report들의 목록이 있어야 합니다.
- 추가한 일지는 `./.journal/INDEX.md` 에 인덱스 형태로 기록

예시:
```
# YYYY-MM-DD {title}

## 어제 요약

...

## 주제 1

...

> 참조: `보고서명`

## 주제 2

...

> 참조: `보고서명`

---
## 참조한 report 목록
2026-03-13-1-reportname1
2026-03-13-2-reportname2
2026-03-13-2-reportname3
---
```

INDEX.md 예시
```
# .journal/ 저널 인덱스
| 보고서 | 내용 |
|--------|------|
| `2026-02-24-journal1` | 짧은 설명1 |
| `2026-02-25-journal2` | 짧은 설명2 |
```
