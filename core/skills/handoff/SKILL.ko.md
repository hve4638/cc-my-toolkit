# handoff

현재 세션에서 수행·논의한 작업 맥락을 작업 단위로 분할해 `.agent-memory/handoff/` 아래에 저장한다. 맥락 0 의 다음 세션 Claude 가 즉시 이어갈 수 있는 형태로 작성한다 (사람용 요약 X).

---

## 저장 규격

- 경로: `.agent-memory/handoff/YYYY-MM-DDThhmm_<title-slug>/`
- 제목 슬러그: kebab-case, 40 자 이내, 한글 허용. 이질적 작업의 혼합이면 `mixed`
- 파일명: `INDEX.md` + `NN-<slug>.md` (2 자리 zero-pad, 예: `01-foo.md`)
- 누적: 덮어쓰기 없음, 매 호출마다 신규 폴더 생성
- 타임스탬프 조회: `date +"%Y-%m-%dT%H%M"` 등으로 현재 시각 조회

---

## 대화 프로토콜

한 번 호출로 즉시 저장 ✗. **1~2 단계만 사용자와 조율**하고, 그룹핑이 확정되면 **3~5 단계는 자동 진행**한다.
1~2 단계 중에는 "이건 빼라", "이 작업은 더 자세히" 같은 지시가 반복될 수 있으며 그때마다 즉시 반영.

### 1. 전수 나열 (사용자 협의)
현재 세션에서 수행·논의한 **모든 작업 맥락을 빠짐없이** 먼저 사용자에게 나열. 누락 방지가 목표.

### 2. 그룹핑 제안 (사용자 협의)
응집도 기준으로 묶음 안을 제시. 합치기·나누기·제외·추가를 사용자 지시에 따라 반복 조정.

**그룹핑이 사용자 확정되면 3~5 단계는 추가 확인 없이 자동 실행한다.**

### 3. 제목 자동 결정
세션의 지배 주제를 나타내는 슬러그 1 개를 자동 선정. 진짜 잡탕이면 `mixed`. 사용자 승인 단계 없음.

### 4. 문서 자동 작성
작업 문서 본문을 자동 작성. 권장 섹션 중 비어 있는 것은 자동 생략.

### 5. 자동 저장
`.agent-memory/handoff/<timestamp>_<title>/` 에 즉시 기록. 별도 최종 승인 단계 없음.

---

## 분할 원칙

- 고정 섹션 구조 없음 → 작업 응집도 기반 동적 분할
- 응집력 있는 작업 묶음 = 1 파일
- 완전히 독립인 작업 = 별도 파일
- 파일 개수가 적절한지의 기준은 "다음 세션 Claude 가 파일 하나 단위로 이해 가능한가"

### 작업 문서 권장 섹션 (느슨, 비면 생략)
- 의도 — 사용자가 원래 하려던 것과 왜
- 지금까지 — 진행 연대기
- 결정과 이유 — 선택지 / 고른 것 / 이유 (롤백 방지)
- 미해결 — 미정 / 블로커 / 답변 대기
- 다음 수 — 즉시 실행 가능한 액션. **세션 마지막에 진행 중이던 작업 1 건에만 기록**한다. 이미 종결된 작업 파일에는 절대 두지 않는다 (pickup 시 끝난 일을 다시 제안하는 사고 방지).

### 작성 규칙
- 코드 블록 복붙 금지
- 참조는 **파일 경로 + 라인 번호 + 의도**만
- "무엇을 했다" 가 아니라 "왜 했다 / 어디까지 왔다 / 다음 수"

## INDEX.md

항상 생성. 다음 세션 Claude 의 진입점.

- 작업 목록 (파일명 + 한 줄 요약). 각 항목에 `[done]` / `[active]` 상태 마커를 붙인다. `[active]` 는 정확히 1 개 (또는 0 개 — 세션이 자연스럽게 종결된 경우).
- 우선순위 / 의존관계
- "어디부터 읽을지" 명시적 안내
- 현재 컨텍스트 — `[active]` 로 표시된 파일 1 개를 명시. **pickup 의 "다음 할 일" 응답이 참조하는 유일한 출처**. 활성 작업이 없으면 `없음 (세션 종결)` 로 명시.

---

## 파일시스템 탐색 금지 (soft)

handoff 는 현재 세션에서 이미 아는 맥락만 정리한다. 세션 외부 정보 혼입 방지가 목적.

예외 (탐색 허용):
- 모호·모순 영역의 사실 확인
- 사용자가 명시적으로 탐색을 지시한 경우

---

## 경계

- inlay — 코드 작성 방법론 가드레일
- handoff — 세션 간 맥락 전달 규칙 (명시적 호출, 일회성)
- MEMORY.md — 프로젝트 영구 지식 (자동)

handoff 에 담은 내용 중 프로젝트 영구 지식이 될 사항은 별도로 memory 에 승격하는 판단을 한다. handoff 자체는 세션 일회성이다.
